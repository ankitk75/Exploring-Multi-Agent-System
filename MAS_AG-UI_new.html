⚡ How MAF, MCP, and AG-UI Work Together

01. The Engine

Microsoft Agent Framework (MAF)

MAF acts as the backend execution environment. Using MagenticBuilder, you define the Orchestrator Agent (the magentic manager) and Participant Agents (Math, String). MAF handles the routing logic, prompting, and maintaining the conversation graph.

02. The Tools

Model Context Protocol (MCP)

Instead of hardcoding functions, MAF integrates with MCP. When the Math Agent needs to multiply, MAF acts as an MCP Client and sends a JSON-RPC request to the connected MCP Server hosting the Calculator tools, standardizing tool execution.

03. The Protocol

AG-UI Protocol

AG-UI is the standardized wire protocol. MAF emits internal framework states. An adapter translates these into strict AG-UI events (e.g., RunStart, MessageStart, ToolCallStart, ToolCallEnd) and streams them to the UI via Server-Sent Events (SSE). Note: There are no arbitrary events like 'AgentThought'.
